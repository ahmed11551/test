# План выполнения тестового задания

## Анализ задачи

### Понятные части
- Требования к технологиям: React + TypeScript для фронтенда, Python/Go/JS для бекенда
- Функциональность: однопользовательский чат с сохранением истории сообщений
- Дизайн: необходимо воспроизвести интерфейс Telegram чата
- Обязательные элементы: время сообщений, сохранение истории после перезагрузки
- Необязательные элементы: эмодзи, скрепка, микрофон, дата, блок пустого чата

### Требуют уточнения
- Детали дизайна: точные цвета, отступы, шрифты (будут определены визуально при реализации)
- Структура API: формат запросов/ответов (спроектирую самостоятельно)
- Деплой: выбор платформы (Vercel + Railway/Render или альтернативы)

## Порядок выполнения

### 1. Планирование и настройка окружения
- Создать структуру проекта (отдельные директории для фронтенда и бекенда)
- Настроить инструменты разработки
- Определить структуру API

### 2. Реализация бекенда
- Настроить FastAPI проект
- Создать эндпоинты для получения и отправки сообщений
- Реализовать хранение сообщений в оперативной памяти (in-memory storage)
- Добавить CORS для работы с фронтендом
- Протестировать API локально

### 3. Реализация фронтенда
- Настроить React + TypeScript проект с Tailwind CSS
- Создать компоненты: список сообщений, поле ввода, контейнер чата
- Реализовать стилизацию в стиле Telegram
- Интегрировать с бекенд API
- Реализовать загрузку истории сообщений при монтировании компонента
- Добавить обработку ошибок

### 4. Деплой
- Задеплоить бекенд на публичный сервис (Railway/Render)
- Задеплоить фронтенд на Vercel или аналогичный сервис
- Настроить CORS для работы между фронтендом и бекендом
- Протестировать работоспособность в продакшене

### 5. Документация и ретроспектива
- Создать файл саморевью с анализом выполненной работы

## Технические решения

### Бекенд
- **Фреймворк**: FastAPI (Python) - быстрый, современный, с автоматической документацией
- **Хранение данных**: In-memory список сообщений (можно расширить до БД при необходимости)
- **API структура**:
  - `GET /messages` - получение всех сообщений
  - `POST /messages` - отправка нового сообщения
  - Формат сообщения: `{ id, text, timestamp, sender }`

### Фронтенд
- **Сборщик**: Vite для быстрой разработки
- **Стилизация**: Tailwind CSS для быстрой стилизации в стиле Telegram
- **Архитектура компонентов**:
  - `ChatContainer` - основной контейнер
  - `MessageList` - список сообщений
  - `MessageItem` - отдельное сообщение
  - `MessageInput` - поле ввода с кнопкой отправки
- **State management**: React hooks (useState, useEffect)

### Деплой
- **Бекенд**: Railway или Render (бесплатные тарифы, простой деплой)
- **Фронтенд**: Vercel (оптимизирован для React, автоматический деплой)
- **CORS**: Настроить на бекенде для работы с фронтендом
